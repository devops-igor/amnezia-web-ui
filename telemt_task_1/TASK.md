# TASK-1: Telemt proxy links use port 443 instead of 18443

## Problem

When creating or copying a Telemt proxy connection link, the port in the
`tg://proxy?server=...&port=443&secret=...` URL defaults to **443**.
The desired default external port is **18443**.

## Root Cause

The default port is propagated through **three** places, all set to 443:

1. **`templates/server.html` line 818** — JS sets the port input to `'443'`
   when telemt is selected, overriding the server-side default.

2. **`telemt_manager.py` line 78** — `install_protocol()` default parameter
   `port="443"`.

3. **`protocol_telemt/config.toml` line 28** — template has
   `public_port = 443` (gets patched on install, but the raw template
   defaults to 443).

There are **zero** references to 18443 in the entire project.

### How the port flows end-to-end

```
UI (server.html) → API (app.py InstallProtocolRequest) → 
  install_protocol(port=443) → 
    patches config.toml: public_port = 443 →
    patches docker-compose.yml: "443:443" → host:container mapping
  stored in server["protocols"]["telemt"]["port"] = "443" →
    add_client(port=443) → link uses &port=443
    get_client_config(port=443) → link uses &port=443
    API-generated links also use public_port=443 from config.toml
```

The internal container port (443 inside Docker) is correct and should NOT
change. Only the **external/host** port and **public_port** (shown in
links) should default to 18443.

## Fix Plan

### Files to change

| # | File | Line | Current | New | Notes |
|---|------|------|---------|-----|-------|
| 1 | `telemt_manager.py` | 78 | `port="443"` | `port="18443"` | Default param for install |
| 2 | `templates/server.html` | 818 | `portInput.value = '443'` | `portInput.value = '18443'` | UI default for telemt |
| 3 | `protocol_telemt/config.toml` | 28 | `public_port = 443` | `public_port = 18443` | Template default (still gets patched) |

### Files NOT to change

| File | Line | Why |
|------|------|-----|
| `protocol_telemt/config.toml` | 32 | `port = 443` is the **internal daemon listen port** — keep as-is |
| `protocol_telemt/docker-compose.yml` | 8 | `"443:443"` is the **internal container port** — gets patched on install |
| `protocol_telemt/Dockerfile` | 78,105 | `EXPOSE 443` documents the **internal** port — keep as-is |
| `telemt_manager.py` | 59 | `docker port ... 443` queries the **internal** container port — correct |
| `telemt_manager.py` | 146 | Substitution `"443:443"` → `"{port}:443"` keeps internal port — correct |

### Additional consideration

- **Existing installations**: Servers already deployed with port 443 will have
  `server["protocols"]["telemt"]["port"] = "443"` stored in their config. Changing
  the defaults only affects **new** installations. Existing deployments keep
  their stored port unless explicitly reconfigured. This is likely acceptable
  since the port mapping in docker-compose and config.toml was already set
  during the original install.

## Acceptance Criteria

- [ ] New Telemt installations default to external port 18443
- [ ] `tg://proxy` links generated by add_client use port 18443 by default
- [ ] API-generated links (from /v1/users) use public_port 18443 by default
- [ ] Internal container port remains 443 (unchanged)
- [ ] Existing installations are not broken (stored port preserved)

## Priority

High — user-facing bug. Wrong port in proxy links = links don't work.
